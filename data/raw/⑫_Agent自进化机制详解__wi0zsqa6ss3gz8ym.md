# ⑫ Agent自进化机制详解

<!-- source: yuque://zhongxian-iiot9/hlyypb/wi0zsqa6ss3gz8ym -->

:::color3
**简介：**本文系统梳理了 **<font style="color:#ED740C;">2025 年至今 Agent 自进化领域的最新研究进展</font>**，深入探讨了 Agent 如何通过经验沉淀、强化学习以及多智能体协同等方式实现自我提升，并总结了该领域的核心技术路线与未来发展趋势。

:::

:::color5
**<font style="color:#601bde;">1. 核心问题与研究背景</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

在过去的认知中，大型语言模型（LLM）在推理阶段缺乏梯度更新机制，这被普遍视为其本质限制。然而，2025 年的各项研究为这一问题提供了全新的解答。这一转变并非源于单一的技术突破，而是**<font style="color:#74B602;">众多研究共同指向了一个核心目标：赋予 Agent 从自身交互经历中持续学习并不断优化的能力。</font>**

在常规的应用场景中，Agent 每日执行大量任务，其成功或失败的交互轨迹往往被直接遗弃。导致在后续任务中，Agent 仍需从零开始，这无疑造成了计算资源与经验数据的极大浪费。

<font style="background-color:#FBDFEF;">自进化研究的核心诉求在于：探索如何将这些历史交互经历有效沉淀，进而转化为更优的执行策略、更丰富的工具集合，亦或是更稳健的模型权重。</font>

:::color5
**<font style="color:#601bde;">2. 两大主要研究路线</font>**

:::

**<font style="color:#74B602;">2025 年的 Agent 自进化研究主要沿着两条核心路径展开：</font>**

+ **推理时进化（不改变基础模型）**：依赖于经验、技能与记忆的积累，在推理阶段实现能力的动态提升。
+ **训练型进化（修改模型权重）**：直接通过强化学习等手段调整模型权重，使模型在持续训练中逐步增强。

此外，学术界也探索了介于两者之间的混合路线，以及基于多智能体协同进化的前沿方向。

# **第一类：经验与技能积累，不改模型权重**
:::color3
**简介：**本类研究侧重于在不修改基础模型权重的前提下，将 Agent 的**<font style="color:#ED740C;">成功交互经验抽象为可复用的“技能”或“原则”</font>**，以提升工程落地可行性与推理效率。

:::

:::color5
**<font style="color:#601bde;">1. CASCADE：材料科学与化学领域的技能库体系</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2512.23880](https://arxiv.org/abs/2512.23880)  
**发表时间：** 2025 年 12 月  
**研究机构：** UC Berkeley

**<font style="color:#74B602;">CASCADE 框架聚焦于材料科学与化学这一高度专业化的领域，其对“技能”概念的界定在同类研究中尤为清晰。</font>**该框架将 Agent 的能力划分为两个层级：底层为基础的工具调用，上层则为对工具调用的高级封装，即“技能”。这些技能具备可执行、可共享以及随时间累积的特性。

系统由两大元技能驱动：

+ **持续学习**：通过网络搜索、代码提取与记忆调用机制，不断获取并整合新技能。
+ **自我反思**：借助内省机制与知识图谱探索，对现有技能进行精炼与优化。

研究团队还构建了 SciSkillBench 基准测试，涵盖 116 个材料科学与化学研究任务。实验结果表明，结合 GPT-5 与 CASCADE 框架，任务成功率高达 93.3%，而缺乏进化机制的基线模型仅为 35.4%，性能提升显著。此外，该技能库支持在不同 Agent 及人类科研人员之间共享，构建了一个团队级的知识积累系统。

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171904040-68ccf986-794a-499d-b183-9ecd7b9dddba.png)

:::color5
**<font style="color:#601bde;">2. STELLA：生物医学领域的动态工具海洋与进化模板库</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2507.02004](https://arxiv.org/abs/2507.02004)  
**发表时间：** 2025 年 7 月  
**研究机构：** Stanford、Princeton

**<font style="color:#74B602;">STELLA 专注于生物医学领域，成功在极度专业化的场景中实现了自进化机制，并取得了详实的数据验证。其核心架构包含两大机制：</font>**

+ **动态工具海洋（Tool Ocean）**：配备独立的工具创建 Agent（Tool Creation Agent），能够自主发现、验证并集成新的生物信息学工具，无需人类手动干预。
+ **进化模板库（Evolution Template Library）**：类似于 EvolveR 的原则库，专门用于存储和精炼从历史经验中提取的推理策略。

在性能表现上，STELLA 在 Humanity’s Last Exam 生物医学子集上达到了约 26% 的准确率；在 LAB-Bench 的 DBQA 子任务和 LitQA 子任务上分别达到 54% 和 63%，超越同期领先模型最高达 6 个百分点。尤为关键的是，随着试验次数的增加，其准确率呈现出近乎翻倍的增长，充分证实了自进化效果的真实性与有效性。

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171925584-4ea5f35c-c758-4c6d-a417-69aa989eb3fa.png)

:::color5
**<font style="color:#601bde;">3. AutoSkill：基于日常交互的技能生命周期管理</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2603.01145](https://arxiv.org/abs/2603.01145)  
**发表时间：** 2026 年 3 月  
**研究机构：** 华东师范大学、上海AI Lab

**<font style="color:#74B602;">AutoSkill 的核心理念在于对技能本身进行全生命周期管理。在 Agent 执行日常任务的过程中，AutoSkill 能够从交互经历中识别出高频出现的模式，并将其抽象为可复用的技能。</font>**

该系统具备动态的技能评估机制：在技能被调用后，系统会持续评估其执行效果，并据此决定对该技能进行精炼、扩展或废弃。当面临新任务时，系统会自动检索相关技能并动态注入至 Agent 的上下文中，彻底免除了人工设计与维护技能库的繁琐工作。这种引入“技能淘汰机制”的设计，有效避免了技能库无限膨胀导致的检索效率与精准度下降问题，在工程实践中极具务实性。

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171373165-23170d48-233e-4ed6-99ea-3d5a611d2835.png)

**<font style="color:#601bde;">4. MemSkill：记忆管理机制的自我进化</font>**

:::

**论文链接：** [https://arxiv.org/abs/2602.02474](https://arxiv.org/abs/2602.02474)  
**发表时间：** 2026 年 2 月  
**研究机构：南洋理工**

**<font style="color:#74B602;">传统的 Agent 记忆系统通常采用静态架构（如固定的向量检索与摘要策略）。MemSkill 突破了这一局限，提出将记忆管理本身视为一种可自我进化的技能。</font>**

该框架将记忆操作（如存储时机、存储内容、检索方式、遗忘策略等）全面纳入技能范畴，使其能够随经验积累而不断优化。当 Agent 面临新型任务时，不仅学习如何完成任务，更在同步学习何种记忆模式最适配该类任务。相较于仅关注“如何利用记忆执行任务”的传统方法，MemSkill 聚焦于“如何进化记忆系统本身”，在任务分布发生偏移时，其高级别的记忆策略优势尤为凸显。

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171391743-6c87eb8f-a71f-433a-844b-baac7dd0f422.png)

**<font style="color:#601bde;">5. SkillWeaver：Web Agent 的自动化技能发现与接口封装</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2504.07079](https://arxiv.org/abs/2504.07079)  
**发表时间：** 2025 年 4 月  
**研究机构：** 俄亥俄州立

**<font style="color:#74B602;">SkillWeaver 创新性地将技能的发现与训练过程转化为类似 API 的构建过程。当 Web Agent 访问新网站时，不仅致力于完成当前任务，还会同步提炼出一个可重复调用的“技能接口”（类似于为该网站自动编写 SDK）。</font>**

在后续面对同一网站或类似场景时，Agent 可直接调用该封装好的技能接口，而无需从头进行逐步推理。这种“Agent 自主编写 API”的模式，对于需要频繁操作同类网站或 SaaS 工具的场景，能够在理论上大幅降低每次任务的 Token 消耗并显著降低出错率。

**<font style="color:#601bde;">6. EvoSkill：基于失败经验的技能挖掘机制</font>**

:::

**论文链接：** [https://arxiv.org/abs/2603.02766](https://arxiv.org/abs/2603.02766)  
**发表时间：** 2026 年 3 月  
**研究机构：** Sentient labs

**<font style="color:#74B602;">与大多数侧重于记录成功轨迹的技能积累框架不同，EvoSkill 专注于从失败案例中挖掘价值。</font>**

在每次任务失败后，EvoSkill 会深度分析失败原因：若是由于缺乏特定能力，则触发新技能的生成机制；若是由于现有技能应用场景错误，则触发对该技能边界条件的修订。这种由失败驱动的发现机制，使得技能库能够有效覆盖那些在常规成功路径中难以触及的边界情况（Corner Cases），弥补了常规学习中的盲区。

<font style="background-color:#FBDFEF;">在多智能体场景中，EvoSkill 还能汇聚不同 Agent 的失败轨迹进行集体分析，从而提炼出单一 Agent 难以独立发现的复杂技能。</font>

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171419570-a0e5cb9b-3c36-41ba-beb5-bbacfa2803cf.png)

**<font style="color:#601bde;">7. Tool-R0：零数据驱动的工具调用学习</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2602.21320](https://arxiv.org/abs/2602.21320)  
**发表时间：** 2026 年 2 月  
**研究机构：** 伊利诺伊大学、苏黎世联邦理工

**<font style="color:#74B602;">Tool-R0 旨在解决新工具引入时的冷启动问题：在缺乏人工整理调用示例的情况下，使 Agent 能够完全自主地从零开始掌握陌生工具的使用方法。</font>**

该框架允许 Agent 自主生成工具调用尝试，并根据实际执行的反馈结果判断正误，进而利用该信号迭代优化调用策略。整个学习过程无需任何预先准备的训练数据，仅需工具具备可执行性并能提供反馈。随着尝试次数的增加，Agent 将逐步建立起对工具参数规范、边界情况及典型用法的深刻理解。相较于 SkillWeaver，Tool-R0 解决的是更为基础的“从零掌握工具调用”的前置问题。

# **第二类：基于强化学习的训练型自进化**
:::color3
**简介：**本类研究通过**<font style="color:#ED740C;">强化学习等技术直接调整模型权重</font>**，使 Agent 在持续的训练与交互中实现底层能力的根本性跃升。

:::

:::color5
**<font style="color:#601bde;">1. EvolveR：从交互轨迹到策略原则的持续闭环</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2510.16079](https://arxiv.org/abs/2510.16079)  
**发表时间：** 2025 年 10 月  
**研究机构：** 浙大、上海AI Lab

**<font style="color:#74B602;">EvolveR 提出了一个逻辑极为清晰的自进化闭环架构，将进化过程拆解为离线蒸馏与在线交互训练两个阶段：</font>**

+ **离线阶段**：在 Agent 完成批量任务后，系统对所有交互轨迹进行提炼，将具体的执行步骤抽象为具备通用性的“策略原则”，并存入原则库。
+ **在线阶段**：Agent 在执行新任务时，实时检索并应用这些原则以指导行动，同时生成新的交互轨迹，反哺下一轮的离线蒸馏训练。

该框架的核心优势在于其抽象层次：它并非简单记录具体的工具调用，而是提炼出诸如“面对多跳问题，先分解子问题再并行检索”的高阶策略，从而赋予技能跨任务的迁移能力。在多跳问答基准测试中，EvolveR 显著优于同类基线模型，且其性能随训练轮次持续提升，展现出极高的成长上限。

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171445584-7fefee28-d160-4c00-b2bd-4f2f00c5180a.png)

:::color5
**<font style="color:#601bde;">2. OpenClaw-RL：基于日常交互状态的强化学习训练</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2603.10165](https://arxiv.org/abs/2603.10165)  
**发表时间：** 2026 年 3 月  
**研究机构：** Princeton（Gen-Verse 实验室）

**<font style="color:#74B602;">OpenClaw-RL 提出了一种极具创新性的训练思路：将 Agent 在日常交互（如与用户对话、调用工具、操作终端或 GUI）中产生的“下一状态”（Next State）直接转化为训练信号。</font>**

该框架将下一状态信号划分为两类：

+ **评估信号**：用于评判当前操作的优劣，通过过程奖励模型（Process Reward Model, PRM）提取为标量奖励。
+ **指导信号**：用于指示更优的操作路径，采用后见之明引导的同策略蒸馏（Hindsight-Guided On-Policy Distillation）方法，从下一状态中提取文本提示，构建增强的教师上下文，进行 Token 级别的方向性优势监督。

<font style="background-color:#FBDFEF;">系统采用完全异步的架构设计，模型服务、PRM 评判与训练器更新并行运作，互不阻塞。这使得 Agent 能够在实际部署和使用过程中实现无缝的持续强化。</font>

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171459500-48c10848-8ea6-4319-a388-af7b26bd2ff3.png)

:::color5
**<font style="color:#601bde;">3. SAGE：技能库与强化学习的深度融合</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2512.17102](https://arxiv.org/abs/2512.17102)  
**发表时间：** 2025 年 12 月  
**研究机构：** 威斯康星大学、AWS

**<font style="color:#74B602;">SAGE 致力于将技能积累（第一类）与强化学习训练（第二类）进行深度融合。</font>**

其核心机制为“连续展开”（Sequential Rollout）：在每次训练展开中，Agent 需连续执行一串相似任务。在早期任务中积累的技能，可直接应用于同一展开序列中的后续任务。这一机制迫使模型在训练过程中不仅要学会完成任务，更要掌握“生成技能”与“复用技能”的能力。同时，系统设计了配套的奖励机制，除了常规的任务完成奖励外，还引入了专门针对技能生成与调用的额外激励信号。

<font style="background-color:#FBDFEF;">在 AppWorld 复杂多应用任务基准测试中，SAGE 相比 GRPO 基线模型，场景目标完成率提升了 8.9 个百分点，交互步数减少 26%，生成 Token 数减少 59%，实现了高准确率与低成本的优异组合。</font>

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171495784-1a9d9009-5ea1-41da-8490-76ff7d6e4175.png)

:::color5
**<font style="color:#601bde;">4. SkillRL：技能库与强化学习策略的双向递归优化</font>**

:::

**论文链接：** [https://arxiv.org/abs/2602.08234](https://arxiv.org/abs/2602.08234)  
**发表时间：** 2026 年 2 月  
**研究机构：** 北卡罗来纳大学

**<font style="color:#74B602;">SkillRL 采用了更为激进的架构设计：实现技能库与强化学习（RL）策略之间的双向递归反馈。</font>**

具体而言，RL 训练生成的交互轨迹会被深度分析，提炼出的新技能将存入技能库；在下一轮 RL 训练中，模型将利用扩充后的技能库生成更高质量的轨迹，进而再次反哺技能提炼过程。每一轮技能库的丰富都有效拓宽了下一轮 RL 的可达策略空间，促使两者协同进化。相较于 SAGE 的“技能辅助训练”单向模式，SkillRL 将技能与策略互为训练数据，理论上具备更高的进化上限。

![](https://cdn.nlark.com/yuque/0/2026/png/29769680/1776171485217-463ed4f0-8b1c-42a3-94f4-0fb306e0740c.png)

:::color5
**<font style="color:#601bde;">5. SE-Agent：基于轨迹级别的自我进化与重组</font>**<font style="color:#D22D8D;">（by草莓师姐）</font>

:::

**论文链接：** [https://arxiv.org/abs/2508.02085](https://arxiv.org/abs/2508.02085)  
**发表时间：** 2025 年 8 月  
**研究机构：** 清华大学、阶跃AI

**<font style="color:#74B602;">针对 Agent 执行错误时通常采用的低效重新采样策略，SE-Agent 提出直接对历史轨迹进行“手术式”改造。</font>**

该框架定义了三种核心轨迹操作：

+ **修订（Revision）**：定位并纠正轨迹中的错误步骤。
+ **重组（Recombination）**：跨轨迹借用并整合成功的子片段。
+ **精炼（Refinement）**：对接近正确的轨迹进行微调打磨，直至完全正确。

与蒙特卡洛树搜索（MCTS）假设路径独立不同，SE-Agent 显式建模了轨迹间的依赖关系，促进了不同轨迹成功经验的相互借鉴。在 SWE-bench Verified 测试中，跨五个主流 LLM 的评估显示，相对基线平均提升最高达 55%，展现出卓越的性能。

# **核心趋势与未来展望**
:::color3
**简介：**综合近一年的前沿研究，Agent 自进化领域在**<font style="color:#ED740C;">数据依赖、奖励机制设计以及安全防御</font>**等方面呈现出明确的发展趋势，为未来的技术突破指明了方向。

:::

:::color5
**<font style="color:#601bde;">1. 零标注数据成为核心驱动力</font>**

:::

摆脱人工标注数据的依赖已成为 2025 年各项新研究的共同追求。无论是通过自生成验证、环境客观反馈，还是多智能体竞争信号，研究者们正致力于构建完全自举的训练闭环。这一趋势不仅是出于降低工程成本的考量，更是实现 Agent 能力规模化扩展的必要前提。

:::color5
**<font style="color:#601bde;">2. 过程奖励机制的必要性</font>**

:::

多项独立研究（如 OpenClaw-RL、SAGE、SE-Agent）均证实：仅依赖最终任务完成状态的稀疏奖励信号，极易导致强化学习训练的不稳定与低效。引入针对 Agent 每一步执行动作的细粒度评估（过程奖励），是确保 RL 训练稳定收敛的核心关键。

:::color5
**<font style="color:#601bde;">3. 奖励模型的同步进化</font>**

:::

MetaClaw 等工程实践明确指出，在策略进化过程中，若奖励模型保持静态冻结，极易触发“奖励作弊”（Reward Hacking）现象，导致进化停滞。因此，推动奖励模型与执行策略的同步动态更新，是防止训练系统崩溃的必要保障。

:::color5
**<font style="color:#601bde;">4. 自进化过程中的安全挑战</font>**

:::

Misevolution 研究深刻表明，自进化不仅是性能优化的技术命题，更是严峻的安全命题。如何在保持 Agent 自主进化活力的同时，系统性防御非预期偏移与安全降级，预计将成为 2026 年学术界与工业界重点攻克的方向。

:::color5
**<font style="color:#601bde;">5. 总结与未来展望</font>**

:::

Agent 在部署后不应处于静止状态，每一次交互数据与失败信号都蕴含着进化的价值，这已成为业界的广泛共识。当前，技能积累路线在工程落地方面展现出极高的友好度；强化学习路线则在能力上限方面极具潜力；多智能体协同则为突破数据瓶颈提供了有效途径。

面向未来，大规模部署中在线 RL 的稳定性保障、Misevolution 的系统性防御机制，以及海量技能库的高效检索策略，仍是亟待解决的核心课题，期待在未来的研究中得到进一步的解
